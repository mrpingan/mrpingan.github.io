I/O重定向

cat往文件写内容

这个命令使用了>表示覆盖，如果是>>则表示追加

cat >foo.sh <<EOF
print "$s was here" "$name"
EOF

foo.sh显示结果如下，发现变量$name被解析了

print "%s was here" ""

避免变量被解析，可以这样做，在EOF之前加了一个****

cat >foo.sh <<\EOF
print "$s was here" "$name"
EOF

这样变量$name就不会被解析了

创建一个管道

mkfifo /tmp/pipe

可以一个程序写，另外一个程序读

创建一个临时文件或者目录

创建临时文件

mktemp
# 指定后缀长度和dirname,X必须要为大写
mktemp /tmp/tmp-XXX
mktemp ~/tmp-XXX

创建临时目录

mktemp -d 
#也可以指定后缀长度和dirname
mktemp -d /tmp/tmp.XXXX
mktemp -d ~/tmp.XXXX

History

从安全角度考虑，禁止history

ln -s /dev/null .bash_history
